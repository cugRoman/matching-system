# 撮合演示系统 - 文件导入功能使用说明

## 新增功能
系统现在支持从txt文件导入订单数据，时间戳支持两种格式：
1. 毫秒时间戳（如：1704067200000）
2. 年/月/日 时:分:秒 格式（如：2024/01/01 08:00:00）

## API接口

### 1. 获取文件格式示例
```
GET /api/file/sample
```
返回文件格式说明和示例。

### 2. 上传文件到内存
```
POST /api/file/upload/memory
Content-Type: multipart/form-data
参数: file (文件)
```
上传txt文件，订单数据只加载到内存中。

### 3. 上传文件到数据库
```
POST /api/file/upload/database
Content-Type: multipart/form-data
参数: file (文件)
```
上传txt文件，订单数据保存到数据库。

### 4. 导入本地文件到内存
```
POST /api/file/import/memory
Content-Type: application/x-www-form-urlencoded
参数: filePath (文件路径)
```
示例：`filePath=orders_with_datetime.txt`

### 5. 导入本地文件到数据库
```
POST /api/file/import/database
Content-Type: application/x-www-form-urlencoded
参数: filePath (文件路径)
```

## 文件格式
```
# 订单导入格式说明
# 每行一个订单，字段用空格分隔
# 格式: 订单号 市场 股票代码 方向(B买/S卖) 数量 价格 股东号 [时间戳]
# 时间戳支持两种格式:
# 1. 毫秒时间戳 (如: 1704067200000)
# 2. 年/月/日 时:分:秒 格式 (如: 2024/01/01 08:00:00)

# 示例订单 (使用年/月/日 时:分:秒 格式)
ORD001 XSHG 600519 B 100 205.00 SH001 2024/01/01 08:00:00
ORD002 XSHG 600519 B 150 202.00 SH002 2024/01/01 08:00:01

# 示例订单 (使用毫秒时间戳)
ORD003 XSHG 600519 S 120 200.00 SH005 1704067204000
```

## 注意事项
1. 时间戳为可选字段，如果不提供则使用当前时间
2. 支持混合格式，同一文件中可以同时使用两种时间格式
3. 文件路径可以是相对路径或绝对路径
4. 上传的文件会保存在`uploads`目录中
5. 注释行以`#`开头，会被自动忽略
