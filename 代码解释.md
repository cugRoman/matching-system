# 股票交易对敲撮合系统 - 代码解释文档

## 1. 项目概述

本项目是一个模拟股票交易对敲撮合系统，用于演示和模拟 A 股市场的竞价撮合机制及对敲风控功能。

### 1.1 核心功能

- **对敲风控**：检测同一股东号同时买卖同一股票，标记为非法交易
- **撮合算法**：实现"价格优先、时间优先"的竞价撮合
- **数据隔离**：演示系统和自动模拟系统数据完全独立

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Spring Boot 3.3.5 + Java 21 |
| 前端 | Vue 3 + TypeScript + Vite 6 |
| 通信 | WebSocket (STOMP) |
| 数据库 | H2 内存数据库 |

   - 调用 matchForSecurity() 处理每支股票

3. **matchForSecurity(String securityId)**：单支股票撮合逻辑

   // 步骤5: 更新交易所挂单

## 5. 核心算法说明

### 5.1 对敲检测算法

```java
// 按股东号分组
Map<String, List<Order>> byHolder = allOrders.stream()
    .collect(Collectors.groupingBy(Order::getShareHolderId));

// 检测同一股东号是否同时存在买卖
for (Map.Entry<String, List<Order>> entry : byHolder.entrySet()) {
    boolean hasBuy = entry.getValue().stream()
        .anyMatch(o -> "B".equals(o.getSide()));
    boolean hasSell = entry.getValue().stream()
        .anyMatch(o -> "S".equals(o.getSide()));
    if (hasBuy && hasSell) {
        // 该股东号所有订单标记为非法
    }
}
```

### 5.2 撮合匹配算法

```java
// 买单队列：价格降序，同价时间升序
PriorityQueue<Order> buyQueue = new PriorityQueue<>(
    Comparator.comparing(Order::getPrice).reversed()
        .thenComparing(Order::getTimestamp)
);

// 卖单队列：价格升序，同价时间升序
PriorityQueue<Order> sellQueue = new PriorityQueue<>(
    Comparator.comparing(Order::getPrice)
        .thenComparing(Order::getTimestamp)
);

// 撮合循环
while (!buyQueue.isEmpty() && !sellQueue.isEmpty()) {
    Order buy = buyQueue.peek();
    Order sell = sellQueue.peek();

    // 价格不匹配，无法成交
    if (buy.getPrice() < sell.getPrice()) {
        break;
    }

    // 成交
    int tradeQty = Math.min(buy.getQty(), sell.getQty());
    double execPrice = sell.getPrice();  // 采用卖方价格
}
```

## 8. 常见问题

1. **CORS 错误**：后端已配置 `originPatterns = "*"`
2. **WebSocket 连接失败**：检查后端是否启动，端口是否正确
3. **数据为0**：检查后端定时任务是否启用
