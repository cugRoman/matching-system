# 建表SQL语句和连接方式

## 数据库表SQL

### 1. 订单表（orders）

存储所有提交的订单（包括新请求和交易所挂单），通过状态字段区分当前状态。

sql

```
CREATE TABLE orders (
    cl_order_id      VARCHAR(64)   NOT NULL COMMENT '订单唯一编号',
    share_holder_id  VARCHAR(32)   NOT NULL COMMENT '股东号',
    security_id      VARCHAR(16)   NOT NULL COMMENT '股票代码',
    side             CHAR(1)       NOT NULL COMMENT '买卖方向 B-买入 S-卖出',
    price            DECIMAL(10,2) NOT NULL COMMENT '委托价格',
    qty              INT           NOT NULL COMMENT '委托数量',
    `timestamp`      BIGINT        NOT NULL COMMENT '订单到达时间（毫秒）',
    status           TINYINT       NOT NULL DEFAULT 0 COMMENT '状态：0-未交易(新请求) 1-未交易并挂载 2-部分交易并挂载 3-已完全成交 4-非法(对敲)',
    created_time     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录创建时间',
    updated_time     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后更新时间',
    PRIMARY KEY (cl_order_id),
    INDEX idx_security_id (security_id),
    INDEX idx_share_holder_id (share_holder_id),
    INDEX idx_status (status),
    INDEX idx_timestamp (`timestamp`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表（所有订单）';
```



### 2. 成交记录表（trade_successes）

存储每一笔成交的详细信息。

sql

```
CREATE TABLE trade_successes (
    exec_id           VARCHAR(64)   NOT NULL COMMENT '成交唯一编号',
    security_id       VARCHAR(16)   NOT NULL COMMENT '股票代码',
    buy_order_id      VARCHAR(64)   NOT NULL COMMENT '买方订单号',
    sell_order_id     VARCHAR(64)   NOT NULL COMMENT '卖方订单号',
    buy_share_holder_id VARCHAR(32) NOT NULL COMMENT '买方股东号',
    sell_share_holder_id VARCHAR(32) NOT NULL COMMENT '卖方股东号',
    exec_qty          INT           NOT NULL COMMENT '成交数量',
    exec_price        DECIMAL(10,2) NOT NULL COMMENT '成交价格',
    exec_time         BIGINT        NOT NULL COMMENT '成交时间（毫秒）',
    created_time      DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录创建时间',
    PRIMARY KEY (exec_id),
    INDEX idx_security_id (security_id),
    INDEX idx_buy_order_id (buy_order_id),
    INDEX idx_sell_order_id (sell_order_id),
    INDEX idx_exec_time (exec_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='撮合成功记录表';
```



### 3. 非法订单表（trade_illegals）

存储因对敲等原因被拒绝的订单。

sql

```
CREATE TABLE trade_illegals (
    cl_order_id      VARCHAR(64)   NOT NULL COMMENT '订单唯一编号',
    share_holder_id  VARCHAR(32)   NOT NULL COMMENT '股东号',
    security_id      VARCHAR(16)   NOT NULL COMMENT '股票代码',
    side             CHAR(1)       NOT NULL COMMENT '买卖方向',
    price            DECIMAL(10,2) NOT NULL COMMENT '委托价格',
    qty              INT           NOT NULL COMMENT '委托数量',
    reject_code      INT           NOT NULL COMMENT '拒绝代码（如1001表示对敲）',
    reject_time      BIGINT        NOT NULL COMMENT '拒绝时间（毫秒）',
    status           TINYINT       NOT NULL DEFAULT 4 COMMENT '固定为4（非法）',
    created_time     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录创建时间',
    PRIMARY KEY (cl_order_id),
    INDEX idx_security_id (security_id),
    INDEX idx_share_holder_id (share_holder_id),
    INDEX idx_reject_time (reject_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='撮合失败（非法）记录表';
```

## Spring Boot 连接配置示例

在 `application.properties` 中添加以下配置（根据实际情况修改数据库地址、用户名、密码）：

properties

```
# 数据源配置
spring.datasource.url=jdbc:mysql://localhost:3306/your_database?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.username=your_username
spring.datasource.password=your_password
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# HikariCP连接池配置（可选）
spring.datasource.hikari.maximum-pool-size=10
spring.datasource.hikari.minimum-idle=5
spring.datasource.hikari.idle-timeout=300000
spring.datasource.hikari.connection-timeout=30000
spring.datasource.hikari.max-lifetime=1800000

# JPA配置（如果使用JPA）
spring.jpa.hibernate.ddl-auto=validate
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL8Dialect
spring.jpa.properties.hibernate.format_sql=true
```



如果使用 `application.yml` 格式，示例如下：

yaml

```

    hibernate:

```

## 说明

- 订单表中 `status` 字段用于区分订单所处阶段，便于撮合算法直接查询待处理订单（如 `status IN (0,1,2)`）。
- 撮合成功后，从订单表更新对应订单状态为 `3`，并插入成交记录到 `trade_successes` 表；非法订单则更新状态为 `4` 并插入 `trade_illegals` 表。
- 所有表均添加了必要的索引以提升查询性能，特别是按股票代码、股东号、时间等字段的检索。
- 时间戳字段使用 `BIGINT` 存储毫秒值，与算法中的 `long` 类型一致。
- 价格字段使用 `DECIMAL(10,2)` 保证精度，避免浮点数误差。

以上设计可直接用于实现撮合核心算法的持久化层。

## 2. Mapper 接口（基于 MyBatis 注解）

### 2.1 OrderMapper

java

```
package com.example.mapper;

import com.example.entity.Order;
import org.apache.ibatis.annotations.*;

import java.util.List;

@Mapper
public interface OrderMapper {

    // 插入新订单
    @Insert("INSERT INTO orders (cl_order_id, share_holder_id, security_id, side, price, qty, `timestamp`, status) " +
            "VALUES (#{clOrderId}, #{shareHolderId}, #{securityId}, #{side}, #{price}, #{qty}, #{timestamp}, #{status})")
    int insert(Order order);

    // 根据订单号查询
    @Select("SELECT * FROM orders WHERE cl_order_id = #{clOrderId}")
    @Results(id = "orderResultMap", value = {
            @Result(column = "cl_order_id", property = "clOrderId"),
            @Result(column = "share_holder_id", property = "shareHolderId"),
            @Result(column = "security_id", property = "securityId"),
            @Result(column = "side", property = "side"),
            @Result(column = "price", property = "price"),
            @Result(column = "qty", property = "qty"),
            @Result(column = "timestamp", property = "timestamp"),
            @Result(column = "status", property = "status")
    })
    Order selectById(String clOrderId);

    // 根据股票代码查询所有状态为0/1/2的订单（待处理订单）
    @Select("SELECT * FROM orders WHERE security_id = #{securityId} AND status IN (0,1,2)")
    @ResultMap("orderResultMap")
    List<Order> selectActiveBySecurityId(String securityId);

    // 更新订单状态和数量（用于部分成交后更新）
    @Update("UPDATE orders SET qty = #{qty}, status = #{status} WHERE cl_order_id = #{clOrderId}")
    int updateQtyAndStatus(Order order);

    // 批量更新订单状态（用于对敲检测后标记非法）
    @Update("<script>" +
            "UPDATE orders SET status = 4 WHERE cl_order_id IN " +
            "<foreach item='id' collection='list' open='(' separator=',' close=')'>#{id}</foreach>" +
            "</script>")
    int batchMarkIllegal(@Param("list") List<String> clOrderIdList);

    // 清空请求列表（删除或标记已处理？根据业务需求，这里演示删除status=0的订单）
    @Delete("DELETE FROM orders WHERE status = 0")
    int deleteRequests();
}
```



### 2.2 TradeSuccessMapper

java

```
package com.example.mapper;

import com.example.entity.TradeSuccess;
import org.apache.ibatis.annotations.*;

import java.util.List;

@Mapper
public interface TradeSuccessMapper {

    // 插入成交记录
    @Insert("INSERT INTO trade_successes (exec_id, security_id, buy_order_id, sell_order_id, " +
            "buy_share_holder_id, sell_share_holder_id, exec_qty, exec_price, exec_time) " +
            "VALUES (#{execId}, #{securityId}, #{buyOrderId}, #{sellOrderId}, " +
            "#{buyShareHolderId}, #{sellShareHolderId}, #{execQty}, #{execPrice}, #{execTime})")
    int insert(TradeSuccess tradeSuccess);

    // 根据股票代码查询成交记录（按时间倒序）
    @Select("SELECT * FROM trade_successes WHERE security_id = #{securityId} ORDER BY exec_time DESC")
    @Results(id = "tradeSuccessResultMap", value = {
            @Result(column = "exec_id", property = "execId"),
            @Result(column = "security_id", property = "securityId"),
            @Result(column = "buy_order_id", property = "buyOrderId"),
            @Result(column = "sell_order_id", property = "sellOrderId"),
            @Result(column = "buy_share_holder_id", property = "buyShareHolderId"),
            @Result(column = "sell_share_holder_id", property = "sellShareHolderId"),
            @Result(column = "exec_qty", property = "execQty"),
            @Result(column = "exec_price", property = "execPrice"),
            @Result(column = "exec_time", property = "execTime")
    })
    List<TradeSuccess> selectBySecurityId(String securityId);

    // 查询最近N条成交记录
    @Select("SELECT * FROM trade_successes ORDER BY exec_time DESC LIMIT #{limit}")
    @ResultMap("tradeSuccessResultMap")
    List<TradeSuccess> selectRecent(@Param("limit") int limit);
}
```

            @Result(column = "qty", property = "qty"),

## 3. MyBatis 配置（application.properties 或 application.yml）

在 `application.properties` 中添加：

properties

```
# MyBatis 配置
mybatis.mapper-locations=classpath:mapper/*.xml   # 如果使用XML则配置，注解方式可不配
mybatis.type-aliases-package=com.example.entity
mybatis.configuration.map-underscore-to-camel-case=true   # 开启驼峰映射，这样@Results可以省略字段映射
```



如果使用 `application.yml`：

yaml

```
mybatis:
  mapper-locations: classpath:mapper/*.xml
  type-aliases-package: com.example.entity
  configuration:
    map-underscore-to-camel-case: true
```



开启 `map-underscore-to-camel-case` 后，MyBatis 会自动将数据库下划线字段映射为 Java 驼峰属性，此时 `@Results` 可以省略（除非有特殊转换需求）。但注意主键字段可能需要额外指定（如果名称不完全匹配）。

## 📡 建立 SSH 隧道

在**你的本地机器**（Windows 可使用 PowerShell、Git Bash 或 CMD；Linux/Mac 直接使用终端）上执行以下命令：

bash

```
ssh -L 33060:127.0.0.1:3306 group4@129.211.187.179 -N
```

- `-L 33060:127.0.0.1:3306`：将本地的 33060 端口转发到远程服务器的 127.0.0.1:3306（即 MySQL 监听的本地地址）。
- `group4@129.211.187.179`：你的服务器登录用户和 IP。
- `-N`：只建立隧道，不执行远程命令（隧道建立后终端会卡住，这是正常的，表示隧道正在工作）。

如果希望隧道在后台运行，可以加上 `-f` 参数：

bash

```
ssh -f -L 33060:127.0.0.1:3306 group4@129.211.187.179 -N
```

输入你的服务器密码后，隧道即建立成功。

> ⚠️ **注意**：如果服务器 SSH 端口不是默认的 22，请加上 `-p <端口号>` 参数。

------

## 🔐 确保 MySQL 用户允许从隧道连接

由于 SSH 隧道最终是从服务器的本地回环地址（127.0.0.1）连接到 MySQL，因此 MySQL 用户必须允许从 `127.0.0.1` 或 `localhost` 连接。

### 1. 🖥️ 在本地连接 MySQL

保持 SSH 隧道运行（终端不要关闭），打开另一个本地终端，使用 MySQL 客户端连接：

bash

```
mysql -h 127.0.0.1 -P 33060 -u tunnel_user -p tunnel_user
```

输入密码后即可进入 MySQL 命令行。

如果本地没有安装 MySQL 客户端，可以用其他数据库工具（如 DBeaver、Navicat、DataGrip）连接，主机填 `127.0.0.1`，端口 `33060`，用户名密码同上。
