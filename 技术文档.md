# 模拟股票交易对敲撮合系统

武汉大学校企合作项目 — 模拟券商委前风控系统，实现股票交易对敲（wash trade）检测与内部撮合。

## 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Spring Boot | 3.5.10 |
| 语言 | Java | 21 |
| 构建工具 | Maven | 3.9+ (内置 Maven Wrapper) |
| 前端框架 | Vue 3 + TypeScript | Vue 3.5, TS 5.9 |
| 前端构建 | Vite | 7.3 |
| 数据库 (开发) | H2 内存数据库 | 自动嵌入 |
| 数据库 (生产) | PostgreSQL | 16 |
| 缓存 (生产) | Redis | 7 |
| API 文档 | SpringDoc OpenAPI | 2.8 |
| 实时通信 | WebSocket (STOMP) | — |

## 环境要求

- **JDK 21**（必须，不支持更低版本）
- **Node.js 18+**（推荐 20 LTS）
- **npm**（随 Node.js 一起安装）
- **Git**
- **IntelliJ IDEA**（推荐 2024.1+，确保支持 Java 21）

> 生产部署还需要 Docker（用于运行 PostgreSQL 和 Redis），但**开发阶段不需要**——后端使用 H2 内存数据库，无需任何外部服务。

## 快速开始

### 1. 克隆仓库

```bash
git clone <仓库地址>
cd Stock-Trade-Matching-System
```

### 2. 后端设置（IntelliJ IDEA）

#### 2.1 确认 JDK 版本

打开终端，验证 Java 版本：

```bash
java -version
# 应该输出: java version "21.x.x"
```

如果版本不对，去 [Oracle JDK 21 下载页](https://www.oracle.com/java/technologies/downloads/#java21) 下载安装，然后：

- 设置环境变量 `JAVA_HOME` 指向 JDK 21 安装目录（如 `C:\Program Files\Java\jdk-21.0.10`）
- 将 `%JAVA_HOME%\bin` 添加到系统 `PATH` 中

#### 2.2 在 IDEA 中打开项目

1. **File → Open** → 选择 `backend/matching-system` 目录
2. IDEA 会自动识别为 Maven 项目，等待索引和依赖下载完成
3. 确认 Project SDK：
   - **File → Project Structure → Project → SDK** → 选择 JDK 21
   - **File → Project Structure → Project → Language Level** → 选择 21
4. 确认 Maven 使用正确的 JDK：
   - **File → Settings → Build → Build Tools → Maven → Importing → JDK for importer** → 选择 21
   - **File → Settings → Build → Build Tools → Maven → Runner → JRE** → 选择 21

#### 2.3 启动后端

**方式 A：通过 IDEA 启动（推荐）**

找到 `MatchingSystemApplication.java`（位于 `com.stocktrade.matchingsystem` 包下），右键 → Run。

**方式 B：通过命令行启动**

```bash
cd backend/matching-system

# Windows
.\mvnw.cmd spring-boot:run

# macOS / Linux
./mvnw spring-boot:run
```

启动成功后你会看到类似输出：

```
Started MatchingSystemApplication in X.XXX seconds
```

#### 2.4 验证后端

- 健康检查：浏览器访问 http://localhost:8080/api/admin/health → 应返回 `{"status":"UP", ...}`
- API 文档：浏览器访问 http://localhost:8080/swagger-ui.html → 查看 Swagger UI
- H2 控制台：http://localhost:8080/h2-console （JDBC URL: `jdbc:h2:mem:stocktrade`, 用户名: `sa`, 密码留空）

### 3. 前端设置

#### 3.1 确认 Node.js 版本

```bash
node -v   # 应该 >= 18
npm -v    # 应该 >= 9
```

如果没有安装，去 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本。

#### 3.2 安装依赖

```bash
cd frontend
npm install
```

#### 3.3 启动前端开发服务器

```bash
npm run dev
```

启动后访问 http://localhost:5173 即可看到系统界面。

> 前端 Vite 开发服务器已配置代理，`/api` 和 `/ws` 请求会自动转发到 `http://localhost:8080`，所以**前后端须同时运行**。

### 4. 同时运行前后端

1. 先启动后端（IDEA 中 Run `MatchingSystemApplication`）
2. 再启动前端（终端中 `cd frontend && npm run dev`）
3. 浏览器访问 http://localhost:5173

## 可用的 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/orders` | 提交委托 |
| GET | `/api/orders` | 查询全部委托 |
| GET | `/api/orders/{clOrderId}` | 查询单笔委托 |
| GET | `/api/admin/health` | 健康检查 |
| GET | `/api/admin/stats` | 系统统计（待完善）|

### 提交委托示例

```bash
curl -X POST http://localhost:8080/api/orders \
  -H "Content-Type: application/json" \
  -d '{
    "clOrderId": "ORD_001",
    "market": "XSHG",
    "securityId": "600519",
    "side": "B",
    "qty": 1000,
    "price": 1800.00,
    "shareHolderId": "SH001",
    "accountId": "ACC001"
  }'
```

PowerShell 中可用：

```powershell
$body = @{
    clOrderId = "ORD_001"
    market = "XSHG"
    securityId = "600519"
    side = "B"
    qty = 1000
    price = 1800.00
    shareHolderId = "SH001"
    accountId = "ACC001"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:8080/api/orders" -Method Post -Body $body -ContentType "application/json"
```

## 常见问题

**Q: 后端启动报 `不支持发行版本 21`**
A: JAVA_HOME 或 PATH 未指向 JDK 21。请检查 `java -version` 输出，以及 IDEA 中 Project SDK 设置。

**Q: 前端页面显示空白**
A: 确认后端正在运行（检查 http://localhost:8080/api/admin/health ），前端需要后端提供数据。

**Q: IDEA 中 Lombok 注解报红**
A: 安装 Lombok 插件（Settings → Plugins → 搜索 Lombok → Install），并开启 Annotation Processing（Settings → Build → Compiler → Annotation Processors → Enable annotation processing）。

**Q: npm install 失败**
A: 尝试删除 `node_modules` 和 `package-lock.json` 后重新 `npm install`。如遇网络问题，可切换为淘宝镜像：`npm config set registry https://registry.npmmirror.com`。
